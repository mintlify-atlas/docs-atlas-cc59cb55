The Ecommerce Backend API is configured via environment variables that override default values in `application.properties`. This guide covers all configuration options organized by component.

## Quick Start

Create a `.env` file or set environment variables for your deployment:

```bash
# Server
SERVER_PORT=8081

# Database
SPRING_DATASOURCE_URL=mysql://your-db-host:3306/ecommerce
SPRING_DATASOURCE_USERNAME=your_username
SPRING_DATASOURCE_PASSWORD=your_password

# JWT
JWT_SECRET=your-secret-key-min-256-bits

# Kafka
KAFKA_BOOTSTRAP=your-kafka-broker:9092
KAFKA_USERNAME=your_kafka_user
KAFKA_PASSWORD=your_kafka_password

# Keycloak
KEYCLOAK_TOKEN_URL=http://keycloak:8080/realms/ecommerce/protocol/openid-connect/token
KEYCLOAK_CLIENT_SECRET=your-client-secret

# Email
SPRING_MAIL_USERNAME=your-email@gmail.com
SPRING_MAIL_PASSWORD=your-app-password

# Middleware
KAFKA_MIDDLEWARE_URL=http://middleware:8090
```

## Configuration Sections

<Accordion title="Server Configuration">

### Server Port

The application runs on port 8081 by default:

```properties
server.port=${SERVER_PORT:8081}
```

**Environment Variable:** `SERVER_PORT`

### Context Path

All endpoints are prefixed with `/api`:

```properties
server.servlet.context-path=/api
server.forward-headers-strategy=framework
```

This means your API will be accessible at `http://localhost:8081/api/*`

### Application Name

```properties
spring.application.name=Ecommerce
```

</Accordion>

<Accordion title="Database Configuration">

### MySQL Connection

The application connects to MySQL using environment variables:

```properties
spring.datasource.url=jdbc:mysql://${SPRING_DATASOURCE_URL}
spring.datasource.username=${SPRING_DATASOURCE_USERNAME}
spring.datasource.password=${SPRING_DATASOURCE_PASSWORD}
```

**Required Environment Variables:**
- `SPRING_DATASOURCE_URL` - MySQL connection URL (format: `host:port/database`)
- `SPRING_DATASOURCE_USERNAME` - Database username
- `SPRING_DATASOURCE_PASSWORD` - Database password

### JPA/Hibernate Settings

```properties
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=false
spring.jpa.properties.hibernate.format_sql=true
```

- **ddl-auto=update**: Automatically updates schema based on entities
- **show-sql=false**: SQL queries not logged (set to `true` for debugging)
- **format_sql=true**: Pretty-prints SQL when logging is enabled

### H2 Database Disabled

```properties
spring.h2.console.enabled=false
spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.h2.H2ConsoleAutoConfiguration
spring.datasource.embedded-database-connection=none
```

H2 is completely disabled for production deployments.

### SQL Initialization

```properties
spring.jpa.defer-datasource-initialization=false
spring.sql.init.mode=never
```

No SQL scripts are executed on startup to prevent accidental data modifications.

</Accordion>

<Accordion title="Kafka Configuration">

### Bootstrap Servers

```properties
spring.kafka.bootstrap-servers=${KAFKA_BOOTSTRAP:localhost:9092}
```

**Environment Variable:** `KAFKA_BOOTSTRAP`

### Consumer Configuration

```properties
spring.kafka.consumer.group-id=ventas-ms
spring.kafka.consumer.auto-offset-reset=earliest
spring.kafka.consumer.value-deserializer=org.springframework.kafka.support.serializer.JsonDeserializer
```

- **group-id**: Consumer group identifier
- **auto-offset-reset**: Start from earliest messages on first connection
- **value-deserializer**: Deserialize JSON messages

### JSON Deserializer Settings

```properties
spring.kafka.consumer.properties.spring.json.use.type.headers=false
spring.kafka.consumer.properties.spring.json.value.default.type=ar.edu.uade.ecommerce.ventas.EventMessage
spring.kafka.consumer.properties.spring.json.trusted.packages=*
```

### SASL Authentication

Kafka uses SCRAM-SHA-512 authentication:

```properties
spring.kafka.properties.security.protocol=SASL_PLAINTEXT
spring.kafka.properties.sasl.mechanism=SCRAM-SHA-512
spring.kafka.properties.sasl.jaas.config=org.apache.kafka.common.security.scram.ScramLoginModule required username="${KAFKA_USERNAME}" password="${KAFKA_PASSWORD}";
```

SASL is configured for:
- **AdminClient** (startup operations)
- **Producer** (publishing events)
- **Consumer** (receiving events)

**Required Environment Variables:**
- `KAFKA_USERNAME` - Kafka authentication username
- `KAFKA_PASSWORD` - Kafka authentication password

### Ventas Consumer

```properties
ventas.kafka.enabled=true
ventas.kafka.topic=ventas
ventas.kafka.concurrency=3
ventas.kafka.listen-ventas=false
```

- **enabled**: Enable Kafka consumer
- **topic**: Topic name for ventas events
- **concurrency**: Number of concurrent consumers (3 threads)
- **listen-ventas**: Disabled by default (only listens to inventario)

### Inventario Consumer

```properties
inventario.kafka.topic=inventario
inventario.kafka.concurrency=1
inventario.kafka.listen-inventario=true
```

The application primarily consumes from the `inventario` topic.

### Error Handling

```properties
ventas.kafka.error.maxAttempts=3
ventas.kafka.error.backoff.ms=500
ventas.kafka.dlq.enabled=false
ventas.kafka.dlq.topicSuffix=.dlq
```

### Missing Topics

```properties
spring.kafka.listener.missing-topics-fatal=false
```

The application won't fail if topics don't exist at startup (useful for development).

</Accordion>

<Accordion title="Keycloak Configuration">

### Token Endpoint

```properties
keycloak.token.url=${KEYCLOAK_TOKEN_URL:http://localhost:8080/realms/ecommerce/protocol/openid-connect/token}
keycloak.client-id=ventas-app
keycloak.client-secret=${KEYCLOAK_CLIENT_SECRET}
keycloak.refresh.enabled=false
```

**Environment Variables:**
- `KEYCLOAK_TOKEN_URL` - OpenID Connect token endpoint
- `KEYCLOAK_CLIENT_SECRET` - Client secret for service account

**Default Client ID:** `ventas-app`

<Note>
Token refresh is disabled (`keycloak.refresh.enabled=false`) to avoid constant retry loops.
</Note>

</Accordion>

<Accordion title="SMTP Email Configuration">

### Gmail SMTP Settings

```properties
spring.mail.host=smtp.gmail.com
spring.mail.port=587
spring.mail.username=${SPRING_MAIL_USERNAME}
spring.mail.password=${SPRING_MAIL_PASSWORD}
spring.mail.properties.mail.smtp.ssl.trust=smtp.gmail.com
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true
spring.mail.properties.mail.smtp.starttls.required=true
```

**Required Environment Variables:**
- `SPRING_MAIL_USERNAME` - Gmail address
- `SPRING_MAIL_PASSWORD` - Gmail app password (not your regular password)

<Warning>
For Gmail, you must use an [App Password](https://support.google.com/accounts/answer/185833), not your regular account password.
</Warning>

</Accordion>

<Accordion title="JWT Configuration">

### Secret Key

```properties
jwt.secret=${JWT_SECRET}
```

**Environment Variable:** `JWT_SECRET`

<Warning>
The JWT secret must be at least 256 bits (32 characters) for HS256 algorithm. Generate a secure random string for production.
</Warning>

Example generation:

```bash
openssl rand -base64 32
```

</Accordion>

<Accordion title="Communication Middleware">

### Core API URL

```properties
core.api.url=${CORE_API_URL:http://localhost:8086}
```

Used for routing and publishing events to Kafka through the communication layer.

### Kafka Middleware

```properties
communication.intermediary.url=${KAFKA_MIDDLEWARE_URL:http://localhost:8090}
```

### ACK Configuration

```properties
communication.intermediary.ack.enabled=true
communication.intermediary.ack.path=/events/{eventId}/ack
communication.intermediary.ack.method=POST
```

Acknowledgments are sent to the middleware after successfully processing events.

</Accordion>

<Accordion title="Event Retry Configuration">

### Retry Scheduler

```properties
ventas.retry.enabled=true
ventas.retry.cron=0 0 */6 * * *
ventas.retry.maxAttempts=5
ventas.retry.cooldown.minutes=30
ventas.retry.batchSize=100
```

- **enabled**: Enable retry scheduler
- **cron**: Runs every 6 hours
- **maxAttempts**: Maximum 5 retry attempts per event
- **cooldown.minutes**: Wait 30 minutes between retries
- **batchSize**: Process up to 100 events per run

</Accordion>
