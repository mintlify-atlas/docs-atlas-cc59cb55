## Overview

The Ecommerce Backend API is a comprehensive microservice built with Spring Boot that powers e-commerce operations for the Ventas (Sales) module. This RESTful API provides a complete solution for managing products, user accounts, shopping carts, orders, and event-driven communication between microservices.

## Who is this for?

This API is designed for:

- **Frontend developers** building e-commerce web or mobile applications
- **Backend developers** integrating with the Ventas microservice ecosystem
- **DevOps engineers** deploying and maintaining the platform
- **System architects** understanding the event-driven architecture

### Additional Features

- **Keycloak Integration**: OAuth2/OIDC authentication support with client credentials flow
- **Event Idempotency**: Duplicate event detection and handling for reliable messaging
- **Inventory Synchronization**: Real-time stock updates via Kafka consumers
- **Review System**: Product ratings and reviews with automatic average calculation
- **Email Notifications**: SMTP-based email system for account verification and password resets
- **Health Monitoring**: Dedicated health check endpoints for service and consumer monitoring

## Architecture Overview

The API follows a layered architecture pattern:

- **Controllers**: REST endpoints handling HTTP requests (`/auth`, `/products`, `/cart`, `/user`)
- **Services**: Business logic implementation with transaction management
- **Repositories**: JPA data access layer with custom queries
- **Entities**: JPA entities mapped to database tables
- **Messaging**: Kafka producers and consumers for event-driven communication
- **Security**: JWT-based authentication and authorization filters

## API Endpoints

The API exposes several controller endpoints:

- `/api/auth` - Authentication (login, register, logout, email verification)
- `/api/products` - Product catalog and filtering
- `/api/cart` - Shopping cart operations
- `/api/user` - User profile management
- `/api/purchases` - Order history and management
- `/api/categories` - Product categories
- `/api/brands` - Product brands
- `/api/events` - Event ingestion from external systems

<Warning>
  The API runs behind a `/api` context path. All endpoints must include this prefix when deployed.
</Warning>

## Event-Driven Communication

The API integrates with multiple Kafka topics:

- **ventas** topic: Consumes sales events from other services
- **inventario** topic: Listens for inventory updates and stock changes
- **Event publishing**: Publishes cart, review, and favorite events to the Core API
