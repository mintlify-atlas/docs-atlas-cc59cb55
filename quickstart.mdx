## Get Started Quickly

This quickstart guide will have you running the Ecommerce Backend API with Keycloak authentication in under 5 minutes using Docker Compose.

<Note>
  This quickstart uses Docker Compose to run Keycloak for authentication. The main API application will run locally on your machine.
</Note>

## Prerequisites

Before you begin, ensure you have:

- **Docker** and **Docker Compose** installed
- **Java 17** or higher installed
- **Maven 3.6+** installed
- **MySQL** database running (or use the configuration below for a local setup)
- **Apache Kafka** running on `localhost:9092` (or configure `KAFKA_BOOTSTRAP` env variable)

<Steps>

### Clone the Repository

First, clone the repository and navigate to the project directory:

```bash
git clone <repository-url>
cd ecommerce-backend
```

### Start Keycloak with Docker Compose

The project includes a `docker-compose.yml` file that sets up Keycloak with a pre-configured realm:

<CodeGroup>
```yaml docker-compose.yml
version: '3.8'
services:
  keycloak:
    image: quay.io/keycloak/keycloak:24.0.4
    environment:
      KEYCLOAK_ADMIN: admin
      KEYCLOAK_ADMIN_PASSWORD: admin
      KC_IMPORT: /opt/keycloak/data/import/realm-export.json
    command: ["start-dev"]
    ports:
      - "8080:8080"
    volumes:
      - ./docker/keycloak/realm-export.json:/opt/keycloak/data/import/realm-export.json:ro
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/realms/ecommerce"]
      interval: 10s
      retries: 10
    restart: unless-stopped
```

```bash Start Keycloak
docker-compose up -d
```
</CodeGroup>

<Note>
  Keycloak will be available at `http://localhost:8080` with credentials `admin/admin`. The realm `ecommerce` and client `ecommerce-module` are automatically imported.
</Note>

### Configure Environment Variables

Create a `.env` file or export these environment variables:

```bash .env
# Server Configuration
SERVER_PORT=8081

# Database Configuration
SPRING_DATASOURCE_URL=localhost:3306/ecommerce
SPRING_DATASOURCE_USERNAME=your_db_user
SPRING_DATASOURCE_PASSWORD=your_db_password

# JWT Secret (use a strong secret in production)
JWT_SECRET=your-secret-key-here-min-256-bits

# Email Configuration (Gmail SMTP)
SPRING_MAIL_USERNAME=your-email@gmail.com
SPRING_MAIL_PASSWORD=your-app-password

# Keycloak Configuration
KEYCLOAK_TOKEN_URL=http://localhost:8080/realms/ecommerce/protocol/openid-connect/token
KEYCLOAK_CLIENT_SECRET=ecommerce-secret

# Kafka Configuration
KAFKA_BOOTSTRAP=localhost:9092
KAFKA_USERNAME=your-kafka-user
KAFKA_PASSWORD=your-kafka-password

# Core API (Communication Service)
CORE_API_URL=http://localhost:8086
KAFKA_MIDDLEWARE_URL=http://localhost:8090
```

<Warning>
  Never commit your `.env` file to version control. Add it to `.gitignore` to keep your secrets safe.
</Warning>

### Build the Application

Build the Spring Boot application using Maven:

```bash
mvn clean package -DskipTests
```

The compiled JAR will be created at `target/app.jar`.

### Run the Application

Start the Spring Boot application:

```bash
java -jar target/app.jar
```

Or use the Maven Spring Boot plugin:

```bash
mvn spring-boot:run
```

The API will start on `http://localhost:8081/api`.

<Note>
  Wait for the startup logs to show "Started EcommerceApplication" before making requests.
</Note>

### Make Your First Request

Now let's test the API by registering a user and logging in:

<CodeGroup>
```bash Register a User
curl -X POST http://localhost:8081/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John",
    "lastname": "Doe",
    "email": "john.doe@example.com",
    "password": "SecurePass123!"
  }'
```

```json Response
{
  "message": "Fuiste registrado satisfactoriamente, revisa tu casilla de correo para ingresar luego el token para activar la cuenta",
  "user": {
    "id": 1,
    "name": "John",
    "lastname": "Doe",
    "email": "john.doe@example.com"
  }
}
```
</CodeGroup>

<Note>
  After registration, you'll need to verify your email using the token sent to your inbox before you can log in.
</Note>

### Verify Email and Login

Verify your email with the token received:

```bash
curl -X POST http://localhost:8081/api/auth/verify-email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john.doe@example.com",
    "token": "your-verification-token"
  }'
```

Then log in to get your JWT token:

<CodeGroup>
```bash Login Request
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john.doe@example.com",
    "password": "SecurePass123!"
  }'
```

```json Response
{
  "success": true,
  "bearer_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "name": "John",
    "lastname": "Doe",
    "email": "john.doe@example.com",
    "addresses": []
  }
}
```
</CodeGroup>

### Fetch Products

Use the JWT token to access protected endpoints:

<CodeGroup>
```bash Get Products
curl -X GET http://localhost:8081/api/products \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```bash Get Product by ID
curl -X GET http://localhost:8081/api/products/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```
</CodeGroup>

</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Keycloak not starting">
    Check that port 8080 is not in use by another application. You can change the port mapping in `docker-compose.yml`:
    
    ```yaml
    ports:
      - "8888:8080"  # Map to different host port
    ```
    
    Then update `KEYCLOAK_TOKEN_URL` to use the new port.
  </Accordion>
  
  <Accordion title="Database connection errors">
    Ensure MySQL is running and the credentials in your `.env` file are correct. Test the connection:
    
    ```bash
    mysql -h localhost -u your_db_user -p ecommerce
    ```
  </Accordion>
  
  <Accordion title="Kafka connection errors">
    Verify Kafka is running on the configured bootstrap server. The application will fail to start if Kafka is unreachable. Check logs for:
    
    ```
    Failed to create AdminClient with bootstrap servers: localhost:9092
    ```
  </Accordion>
  
  <Accordion title="Port 8081 already in use">
    Change the `SERVER_PORT` environment variable to use a different port:
    
    ```bash
    export SERVER_PORT=8082
    ```
  </Accordion>
</AccordionGroup>
