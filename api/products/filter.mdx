## Overview

This endpoint allows filtering products by multiple criteria including categories, brands, and price ranges. Results can be sorted and paginated.

## Request Body

<ParamField body="categoryCode" type="integer">
  Filter by single category code
</ParamField>

<ParamField body="categoryCodes" type="array">
  Filter by multiple category codes (takes precedence over single categoryCode)
</ParamField>

<ParamField body="brandCode" type="integer">
  Filter by single brand code
</ParamField>

<ParamField body="brandCodes" type="array">
  Filter by multiple brand codes (takes precedence over single brandCode)
</ParamField>

<ParamField body="priceMin" type="float">
  Minimum price filter
</ParamField>

<ParamField body="priceMax" type="float">
  Maximum price filter
</ParamField>

<ParamField body="sortBy" type="string">
  Sort field: `price` or `relevance` (relevance sorts by calification)
</ParamField>

<ParamField body="sortOrder" type="string">
  Sort direction: `asc` or `desc`
</ParamField>

<ParamField body="page" type="integer" default="0">
  Page number (zero-based)
</ParamField>

<ParamField body="size" type="integer" default="20">
  Number of items per page
</ParamField>

## Response

Returns a paginated list of products matching the filter criteria.

<ResponseField name="content" type="array">
  Array of ProductDTO objects matching the filters
</ResponseField>

<ResponseField name="pageable" type="object">
  Pagination information
  
  <ResponseField name="pageNumber" type="integer">
    Current page number
  </ResponseField>
  
  <ResponseField name="pageSize" type="integer">
    Items per page
  </ResponseField>
</ResponseField>

<ResponseField name="totalElements" type="long">
  Total number of products matching filters
</ResponseField>

<ResponseField name="totalPages" type="integer">
  Total number of pages
</ResponseField>

## Example Requests

<CodeGroup>

```bash Filter by Brand
curl -X POST "https://api.example.com/products/filter" \
  -H "Content-Type: application/json" \
  -d '{
    "brandCode": 101,
    "sortBy": "price",
    "sortOrder": "asc",
    "page": 0,
    "size": 20
  }'
```

```bash Filter by Price Range
curl -X POST "https://api.example.com/products/filter" \
  -H "Content-Type: application/json" \
  -d '{
    "priceMin": 50.0,
    "priceMax": 500.0,
    "sortBy": "relevance",
    "sortOrder": "desc"
  }'
```

```bash Multiple Filters
curl -X POST "https://api.example.com/products/filter" \
  -H "Content-Type: application/json" \
  -d '{
    "categoryCodes": [1, 2, 3],
    "brandCodes": [101, 102],
    "priceMin": 100.0,
    "priceMax": 1000.0,
    "sortBy": "price",
    "sortOrder": "asc",
    "page": 0,
    "size": 20
  }'
```

```javascript JavaScript
const filterRequest = {

const response = await fetch('https://api.example.com/products/filter', {

const data = await response.json();
```

## Filtering Logic

### Category Filtering
- If `categoryCodes` array is provided, it takes precedence over single `categoryCode`
- Products must have at least one category matching the filter
- Only active categories are considered

### Brand Filtering
- If `brandCodes` array is provided, it takes precedence over single `brandCode`
- Products must match the specified brand(s)
- Only active brands are considered

### Price Filtering
- `priceMin`: Products with price greater than or equal to minimum
- `priceMax`: Products with price less than or equal to maximum
- Both filters can be combined

### Sorting
- **price**: Sort by product price
- **relevance**: Sort by product rating (calification)
- Default sort order is ascending if not specified

## Notes

- Only active products are included in results
- Products without prices are excluded when price filters are applied
- Page numbers are zero-based (first page is 0)
- Default page size is 20 items
- All filters are combined with AND logic
