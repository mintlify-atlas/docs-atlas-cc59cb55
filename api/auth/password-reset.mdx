---
title: "Password Reset"
api: "POST /password/*"
description: "Request, validate, and complete password reset"
---

The password reset flow consists of three endpoints to securely reset a user's password.

## Request Password Reset

### POST /password/request

Initiates the password reset process by sending a reset code to the user's email.

#### Request Body

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

#### Response

<ResponseField name="message" type="string">
  Confirmation that a reset code has been sent
</ResponseField>

#### Example

<CodeGroup>
```bash curl
curl -X POST https://api.example.com/password/request \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com"
  }'
```

```json Response (200 OK)
{
  "message": "Se ha enviado un correo con el código de recuperación"
}
```
</CodeGroup>

#### Error Responses

<ResponseField name="404 Not Found">
  Returned when the email doesn't belong to any registered account.
  
  ```json
  {
    "error": "El email no pertenece a ninguna cuenta"
  }
  ```
</ResponseField>

---

## Validate Reset Token

### POST /password/validate

Validates that the reset token is correct before allowing password change.

#### Request Body

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="token" type="string" required>
  Reset token sent to the user's email
</ParamField>

#### Response

<ResponseField name="message" type="string">
  Confirmation that the token is valid
</ResponseField>

#### Example

<CodeGroup>
```bash curl
curl -X POST https://api.example.com/password/validate \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "token": "123456"
  }'
```

```json Response (200 OK)
{
  "message": "Token válido"
}
```
</CodeGroup>

#### Error Responses

<ResponseField name="400 Bad Request">
  Returned when the token is invalid or incorrect.
  
  ```json
  {
    "error": "El token enviado no es correcto"
  }
  ```
</ResponseField>

---

## Change Password

### POST /password/change

Completes the password reset by setting a new password.

#### Request Body

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="token" type="string" required>
  Valid reset token
</ParamField>

<ParamField body="newPassword" type="string" required>
  New password for the account
</ParamField>

#### Response

<ResponseField name="message" type="string">
  Confirmation that the password was changed successfully
</ResponseField>

#### Example

<CodeGroup>
```bash curl
curl -X POST https://api.example.com/password/change \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "token": "123456",
    "newPassword": "newSecurePassword456"
  }'
```

```json Response (200 OK)
{
  "message": "Contraseña cambiada exitosamente"
}
```
</CodeGroup>

#### Error Responses

<ResponseField name="400 Bad Request">
  Returned when the token is invalid or incorrect.
  
  ```json
  {
    "error": "El token enviado no es correcto"
  }
  ```
</ResponseField>

<Warning>
  The reset token can only be used once. After a successful password change, the token becomes invalid.
</Warning>

<Note>
  For security, all three endpoints should be called in sequence: request → validate → change.
</Note>
