---
title: "Delete Cart"
api: "DELETE /cart/{id}"
---

Deletes a cart by its ID. This operation is permanent and cannot be undone. When a cart is deleted, all associated cart items are also removed due to cascade delete rules.

<Note>
  Deleting a cart does NOT automatically restore product stock. Stock restoration only happens when a purchase is cancelled via `DELETE /purchase/{id}`.
</Note>

## Authentication

This endpoint requires a valid Bearer token in the Authorization header.

<ParamField path="id" type="integer" required>
  The unique identifier of the cart to delete
</ParamField>

## Request Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Example: `Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
</ParamField>

## Response

Returns an empty response body on successful deletion.

## Status Codes

- **200 OK** - Cart deleted successfully
- **401 Unauthorized** - Invalid or missing authentication token, or user session is not active
- **404 Not Found** - Cart with the specified ID does not exist

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.example.com/cart/1" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

```javascript JavaScript
const response = await fetch('https://api.example.com/cart/1', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN_HERE'
  }
});

if (response.ok) {
  console.log('Cart deleted successfully');
}
```

```python Python
import requests

response = requests.delete(
    'https://api.example.com/cart/1',
    headers={'Authorization': 'Bearer YOUR_TOKEN_HERE'}
)

if response.status_code == 200:
    print('Cart deleted successfully')
```
</CodeGroup>

## Important Notes

### Stock Restoration

Deleting a cart alone does **NOT** restore product stock. To properly cancel a cart and restore stock, you should:

1. Delete the associated **purchase** using `DELETE /purchase/{id}`, which will:
   - Restore product stock automatically
   - Delete the purchase record
   
2. Or let the purchase expire naturally after 4 hours

### Cascade Deletion

When a cart is deleted:
- All **cart items** are automatically deleted (orphan removal)
- The cart is removed from the database
- Any associated purchases may become orphaned (depending on constraints)

### Recommended Pattern

Instead of deleting the cart directly, consider:

```javascript
// Recommended: Cancel the purchase (restores stock)
await fetch('https://api.example.com/purchase/42', {
  method: 'DELETE',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
});

// Not recommended: Delete cart directly (doesn't restore stock)
await fetch('https://api.example.com/cart/1', {
  method: 'DELETE',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
});
```

## Use Cases

### When to Use DELETE /cart/{id}
- Cleaning up orphaned carts
- Administrative cart removal
- Testing and development

### When to Use DELETE /purchase/{id}
- User cancels their order
- Purchase expires
- Stock needs to be restored
