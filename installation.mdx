## Overview

This guide covers the complete installation process for the Ecommerce Backend API, from prerequisites to running the application in development and production environments.

## Prerequisites

Before installing the Ecommerce Backend API, ensure you have the following installed:

<Steps>

### Java 17

The application requires Java 17 or higher.

<CodeGroup>
```bash Verify Java Version
java -version
```

```bash Install Java 17 (Ubuntu/Debian)
sudo apt update
sudo apt install openjdk-17-jdk
```

```bash Install Java 17 (macOS with Homebrew)
brew install openjdk@17
```
</CodeGroup>

### Maven 3.6+

Maven is required to build the project.

<CodeGroup>
```bash Verify Maven Version
mvn -version
```

```bash Install Maven (Ubuntu/Debian)
sudo apt update
sudo apt install maven
```

```bash Install Maven (macOS with Homebrew)
brew install maven
```
</CodeGroup>

### MySQL Database

The application uses MySQL as its primary database.

<CodeGroup>
```bash Install MySQL (Ubuntu/Debian)
sudo apt update
sudo apt install mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql
```

```bash Install MySQL (macOS with Homebrew)
brew install mysql
brew services start mysql
```

```bash Create Database
mysql -u root -p

CREATE DATABASE ecommerce CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'ecommerce_user'@'localhost' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON ecommerce.* TO 'ecommerce_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```
</CodeGroup>

<Note>
  For development and testing, the application can use H2 in-memory database. However, MySQL is required for production deployments.
</Note>

### Apache Kafka

Kafka is required for event-driven communication between microservices.

<CodeGroup>
```bash Install Kafka (with Docker)
docker run -d --name kafka \
  -p 9092:9092 \
  -e KAFKA_ZOOKEEPER_CONNECT=zookeeper:2181 \
  -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 \
  -e KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1 \
  confluentinc/cp-kafka:latest
```

```bash Verify Kafka is Running
docker ps | grep kafka
```
</CodeGroup>

<Warning>
  The application requires Kafka with SASL authentication (SCRAM-SHA-512). Ensure your Kafka cluster is configured with the appropriate security settings.
</Warning>

</Steps>

## Installation Steps

<Steps>

### Clone the Repository

Clone the Ecommerce Backend API repository:

```bash
git clone <repository-url>
cd ecommerce-backend
```

### Configure Application Properties

The application uses environment variables for configuration. Create a `.env` file or export the following variables:

<CodeGroup>
```bash .env File
# Server Configuration
SERVER_PORT=8081

# Database Configuration
SPRING_DATASOURCE_URL=localhost:3306/ecommerce
SPRING_DATASOURCE_USERNAME=ecommerce_user
SPRING_DATASOURCE_PASSWORD=your_secure_password

# JWT Configuration
JWT_SECRET=your-jwt-secret-key-minimum-256-bits-long

# Email Configuration (Gmail SMTP)
SPRING_MAIL_USERNAME=your-email@gmail.com
SPRING_MAIL_PASSWORD=your-gmail-app-password

# Keycloak Configuration
KEYCLOAK_TOKEN_URL=http://localhost:8080/realms/ecommerce/protocol/openid-connect/token
KEYCLOAK_CLIENT_SECRET=your-keycloak-client-secret

# Kafka Configuration
KAFKA_BOOTSTRAP=localhost:9092
KAFKA_USERNAME=your-kafka-username
KAFKA_PASSWORD=your-kafka-password

# Core API URLs
CORE_API_URL=http://localhost:8086
KAFKA_MIDDLEWARE_URL=http://localhost:8090
```

```bash Export Environment Variables
export SERVER_PORT=8081
export SPRING_DATASOURCE_URL=localhost:3306/ecommerce
export SPRING_DATASOURCE_USERNAME=ecommerce_user
export SPRING_DATASOURCE_PASSWORD=your_secure_password
export JWT_SECRET=your-jwt-secret-key-minimum-256-bits-long
export SPRING_MAIL_USERNAME=your-email@gmail.com
export SPRING_MAIL_PASSWORD=your-gmail-app-password
export KEYCLOAK_TOKEN_URL=http://localhost:8080/realms/ecommerce/protocol/openid-connect/token
export KEYCLOAK_CLIENT_SECRET=your-keycloak-client-secret
export KAFKA_BOOTSTRAP=localhost:9092
export KAFKA_USERNAME=your-kafka-username
export KAFKA_PASSWORD=your-kafka-password
export CORE_API_URL=http://localhost:8086
export KAFKA_MIDDLEWARE_URL=http://localhost:8090
```
</CodeGroup>

#### Key Configuration Parameters

Here's what each configuration parameter does:

| Parameter | Description | Default |
|-----------|-------------|----------|
| `SERVER_PORT` | Port on which the API runs | `8081` |
| `SPRING_DATASOURCE_URL` | MySQL database connection URL | Required |
| `SPRING_DATASOURCE_USERNAME` | Database username | Required |
| `SPRING_DATASOURCE_PASSWORD` | Database password | Required |
| `JWT_SECRET` | Secret key for signing JWT tokens (min 256 bits) | Required |
| `SPRING_MAIL_USERNAME` | SMTP email username for sending emails | Required |
| `SPRING_MAIL_PASSWORD` | SMTP email password or app password | Required |
| `KEYCLOAK_TOKEN_URL` | Keycloak token endpoint URL | `http://localhost:8080/realms/ecommerce/protocol/openid-connect/token` |
| `KEYCLOAK_CLIENT_SECRET` | Keycloak client secret for service authentication | Required |
| `KAFKA_BOOTSTRAP` | Kafka bootstrap servers | `localhost:9092` |
| `KAFKA_USERNAME` | Kafka SASL username | Required |
| `KAFKA_PASSWORD` | Kafka SASL password | Required |
| `CORE_API_URL` | Core communication API base URL | `http://localhost:8086` |
| `KAFKA_MIDDLEWARE_URL` | Kafka middleware service URL | `http://localhost:8090` |

<Warning>
  **Security Best Practices:**
  - Never commit `.env` files to version control
  - Use strong, randomly generated secrets for `JWT_SECRET`
  - Use Gmail App Passwords instead of your account password for `SPRING_MAIL_PASSWORD`
  - Rotate credentials regularly in production environments
</Warning>

### Review Application Properties

The `src/main/resources/application.properties` file contains the application configuration:

<CodeGroup>
```properties Key Settings
# Application Name
spring.application.name=Ecommerce

# Server Port (overrideable via environment)
server.port=${SERVER_PORT:8081}

# Context Path (all endpoints prefixed with /api)
server.servlet.context-path=/api

# Database Configuration
spring.datasource.url=jdbc:mysql://${SPRING_DATASOURCE_URL}
spring.datasource.username=${SPRING_DATASOURCE_USERNAME}
spring.datasource.password=${SPRING_DATASOURCE_PASSWORD}

# JPA/Hibernate Settings
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=false

# Kafka Consumer Configuration
spring.kafka.bootstrap-servers=${KAFKA_BOOTSTRAP:localhost:9092}
spring.kafka.consumer.group-id=ventas-ms
spring.kafka.consumer.auto-offset-reset=earliest

# Kafka Topics
ventas.kafka.topic=ventas
inventario.kafka.topic=inventario
```

```properties SASL Authentication
# Kafka SASL Configuration (SCRAM-SHA-512)
spring.kafka.properties.security.protocol=SASL_PLAINTEXT
spring.kafka.properties.sasl.mechanism=SCRAM-SHA-512
spring.kafka.properties.sasl.jaas.config=org.apache.kafka.common.security.scram.ScramLoginModule required username="${KAFKA_USERNAME}" password="${KAFKA_PASSWORD}";
```
</CodeGroup>

<Note>
  The application uses `spring.jpa.hibernate.ddl-auto=update` which automatically creates and updates database tables based on JPA entities. In production, consider using Flyway or Liquibase for database migrations.
</Note>

### Build the Application

Build the application using Maven:

<CodeGroup>
```bash Build with Tests
mvn clean package
```

```bash Build without Tests (faster)
mvn clean package -DskipTests
```

```bash Build and Install to Local Maven Repository
mvn clean install
```
</CodeGroup>

The compiled JAR will be created at `target/app.jar` (as configured in `pom.xml`).

<Note>
  The build includes JaCoCo code coverage reports which are generated in `target/site/jacoco/`.
</Note>

### Run the Application

You have several options to run the application:

<CodeGroup>
```bash Run JAR Directly
java -jar target/app.jar
```

```bash Run with Maven Spring Boot Plugin
mvn spring-boot:run
```

```bash Run with Custom JVM Options
java -Xmx512m -Xms256m -jar target/app.jar
```

```bash Run with Specific Profile
java -jar target/app.jar --spring.profiles.active=dev
```
</CodeGroup>

### Verify Installation

Once the application starts, verify it's running correctly:

<CodeGroup>
```bash Check Application Status
curl http://localhost:8081/api/ping
```

```bash Access Swagger UI
open http://localhost:8081/api/swagger-ui.html
```

```bash Check Health Endpoint
curl http://localhost:8081/api/ventas/health
```
</CodeGroup>

You should see logs indicating:

```log
Started EcommerceApplication in X.XXX seconds
Tomcat started on port(s): 8081 (http) with context path '/api'
```

</Steps>

## Keycloak Setup

The application integrates with Keycloak for OAuth2/OIDC authentication:

<Steps>

### Start Keycloak with Docker Compose

```bash
docker-compose up -d
```

This starts Keycloak on `http://localhost:8080` with:
- Admin credentials: `admin/admin`
- Pre-configured realm: `ecommerce`
- Client ID: `ventas-app`
- Client secret: configured via `KEYCLOAK_CLIENT_SECRET`

### Access Keycloak Admin Console

1. Navigate to `http://localhost:8080`
2. Log in with `admin/admin`
3. Select the `ecommerce` realm
4. Configure clients, users, and roles as needed

### Token Endpoint

The token endpoint URL is:

```
http://localhost:8080/realms/ecommerce/protocol/openid-connect/token
```

</Steps>

## Production Deployment

For production deployments, consider the following:

<Steps>

### Use External Configuration

Use Spring Cloud Config or external configuration files instead of environment variables:

```bash
java -jar target/app.jar --spring.config.location=file:/etc/ecommerce/application.properties
```

### Enable HTTPS

Configure SSL/TLS certificates:

```properties
server.ssl.key-store=classpath:keystore.p12
server.ssl.key-store-password=your-keystore-password
server.ssl.key-store-type=PKCS12
server.ssl.key-alias=tomcat
```

### Database Connection Pooling

Configure HikariCP for optimal database performance:

```properties
spring.datasource.hikari.maximum-pool-size=10
spring.datasource.hikari.minimum-idle=5
spring.datasource.hikari.connection-timeout=30000
```

### Logging Configuration

Set appropriate log levels for production:

```properties
logging.level.root=WARN
logging.level.ar.edu.uade.ecommerce=INFO
logging.level.org.springframework.kafka=WARN
```

### Monitoring and Observability

Enable Spring Boot Actuator for monitoring:

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection refused">
    **Problem:** Application fails to connect to MySQL
    
    **Solution:**
    - Verify MySQL is running: `sudo systemctl status mysql`
    - Check connection string format: `jdbc:mysql://localhost:3306/ecommerce`
    - Verify credentials and database exists
    - Check firewall rules if using remote database
  </Accordion>
  
  <Accordion title="Kafka consumer errors">
    **Problem:** KafkaConsumer not receiving messages
    
    **Solution:**
    - Verify Kafka is running and accessible
    - Check bootstrap servers configuration
    - Verify SASL credentials are correct
    - Check topic exists: `kafka-topics --list --bootstrap-server localhost:9092`
    - Review consumer group status: `kafka-consumer-groups --describe --group ventas-ms`
  </Accordion>
  
  <Accordion title="JWT token validation errors">
    **Problem:** Authentication fails with invalid token errors
    
    **Solution:**
    - Verify `JWT_SECRET` is set and matches across all instances
    - Ensure secret is at least 256 bits (32 characters)
    - Check token expiration settings
    - Verify clock synchronization across servers
  </Accordion>
  
  <Accordion title="Email sending failures">
    **Problem:** Email verification tokens not being sent
    
    **Solution:**
    - Verify SMTP credentials are correct
    - For Gmail, use an App Password instead of your account password
    - Enable "Less secure app access" in Gmail settings (not recommended)
    - Check firewall allows outbound connections on port 587
    - Review email logs: `logging.level.org.springframework.mail=DEBUG`
  </Accordion>
  
  <Accordion title="Port already in use">
    **Problem:** Cannot start application, port 8081 in use
    
    **Solution:**
    - Find process using port: `lsof -i :8081` (Linux/macOS) or `netstat -ano | findstr :8081` (Windows)
    - Kill the process or change the port:
      ```bash
      export SERVER_PORT=8082
      java -jar target/app.jar
      ```
  </Accordion>
  
  <Accordion title="Hibernate schema validation errors">
    **Problem:** Schema validation errors on startup
    
    **Solution:**
    - Drop and recreate database for clean start (development only):
      ```sql
      DROP DATABASE ecommerce;
      CREATE DATABASE ecommerce;
      ```
    - Or temporarily set: `spring.jpa.hibernate.ddl-auto=create` (WARNING: destroys data)
    - For production, use proper database migration tools
  </Accordion>
</AccordionGroup>
